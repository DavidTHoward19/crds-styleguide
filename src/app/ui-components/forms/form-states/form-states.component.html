States

Validation states are a great way to provide visual feedback when filling out a form. Angular can use the ngModel directive to track the state and validity of the form.

Documentation for Angular 2 form validation can be found here.

Input Field — Valid

A class of ng-valid can only be seen when the input field has the classes of ng-dirty and ng-touched, and the validation criteria is met.

Input Field — Invalid

There are two styles for invalid input states. When the user clicks submit without attempting to enter text in the input field, a class of ng-touched will be added to the input field.


When a user has attempted to enter text in the input field, a class of ng-dirty is added to the input field. This style comes into play when the entered text doesn't meet the validation criteria.
